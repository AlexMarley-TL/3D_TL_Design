  **2. Dynamically import model-viewer in ARButton.tsx:**

  Instead of importing `@google/model-viewer` at the top of ARButton.tsx, use a dynamic import inside a useEffect that runs on mount. This defers the ~150 KB model-viewer package until a page with an AR button is actually rendered.

  ```typescript
  useEffect(() => {
    import('@google/model-viewer').catch(() => {
      // model-viewer failed to load, AR button will remain hidden
    })
  }, [])
  ```

  Keep the rest of ARButton.tsx logic unchanged. The model-viewer custom element registers itself globally when imported, so the `<model-viewer>` JSX element will work once the import resolves.

  Add build.rolldownOptions to split three.js and react into separate vendor chunks so they cache independently:

  ```typescript
        },
      },
    },
  })
  ```

  The goal: initial JS chunk (the entry point) should be under 500 KB gzipped total when all vendor + app chunks are summed. The key win is that model-viewer and lazy pages load on demand.

  **2. Create scripts/compress-models.ts:**

  A Node.js script that:
  - Reads all .glb files from `public/models/` (excluding the `backup/` subdirectory)
  - For each .glb, runs gltf-pipeline's `processGlb` with Draco compression enabled
  - Backs up originals to `public/models/backup/` (create dir if missing, skip if backup already exists for that filename)
  - Writes compressed .glb back to the original path
  - Logs before/after file sizes for each model
  - Reports total savings

  Key implementation:

  ```typescript
  import { readFileSync, writeFileSync, mkdirSync, existsSync, readdirSync, statSync } from 'fs'
  import { join, basename } from 'path'
  // gltf-pipeline is CJS, use createRequire or dynamic import
  import { createRequire } from 'module'
  const require = createRequire(import.meta.url)
  const gltfPipeline = require('gltf-pipeline')
  const processGlb = gltfPipeline.processGlb

  const MODELS_DIR = join(process.cwd(), 'public', 'models')
  const BACKUP_DIR = join(MODELS_DIR, 'backup')
  ```

  The Draco options: `{ dracoOptions: { compressionLevel: 7 } }`

  **3. Add npm script to package.json:**

  Add to scripts: `"compress-models": "npx tsx scripts/compress-models.ts"`

  This uses tsx to run the TypeScript script directly without compilation.
