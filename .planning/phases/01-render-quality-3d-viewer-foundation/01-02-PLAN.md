**Create `src/viewer/TrophyModel.tsx`:**

This component loads a .glb model via drei `useGLTF` and applies a MeshPhysicalMaterial preset to all meshes via scene traversal.

```typescript
import { useEffect, useMemo } from 'react'
import { useGLTF } from '@react-three/drei'
import * as THREE from 'three'
import { METAL_PRESETS } from '../materials/metalPresets'
import type { MetalPreset } from '../types'

interface TrophyModelProps {
  path: string
  preset: MetalPreset
}

export function TrophyModel({ path, preset }: TrophyModelProps) {
  const { scene } = useGLTF(path)

  // Create material once per preset change, not per render
  const material = useMemo(
    () => new THREE.MeshPhysicalMaterial({ ...METAL_PRESETS[preset] }),
    [preset]
  )

  // Apply material to all meshes in the loaded scene
  useEffect(() => {
    scene.traverse((child) => {
      if ((child as THREE.Mesh).isMesh) {
        (child as THREE.Mesh).material = material
      }
    })
  }, [scene, material])

  // Dispose material on unmount
  useEffect(() => {
    return () => { material.dispose() }
  }, [material])

  return <primitive object={scene} />
}
```

Key points:
- `useMemo` for material creation prevents GC spikes from recreating materials each render (Pitfall 6 from research)
- Material is disposed on unmount to prevent WebGL memory leaks
- `metalness: 1.0` is already baked into every METAL_PRESETS entry, so no override needed
- Per D-01, the test model is SheenChair.glb; per D-02, the component accepts any path for easy model swapping

**Create `src/viewer/TrophyViewer.tsx`:**

This is the persistent R3F Canvas wrapper. It is mounted ONCE at the layout level in App.tsx (per D-03, D-04) and NEVER unmounts. Scene content is swapped by changing props.

```typescript
import { Canvas } from '@react-three/fiber'
import { Suspense } from 'react'
import { TrophyScene } from './TrophyScene'
import { LoadingSpinner } from '../components/LoadingSpinner'
import type { MetalPreset } from '../types'

interface TrophyViewerProps {
  modelPath: string | null
  metalPreset: MetalPreset
}

export function TrophyViewer({ modelPath, metalPreset }: TrophyViewerProps) {
  return (
    <div className="trophy-viewer">
      <Canvas
        flat
        dpr={[1, 2]}
        gl={{ antialias: true }}
        camera={{ position: [0, 1, 4], fov: 45 }}
      >
        <Suspense fallback={null}>
          {modelPath && (
            <TrophyScene modelPath={modelPath} metalPreset={metalPreset} />
          )}
        </Suspense>
      </Canvas>
      {/* DOM-based loading spinner overlaid on Canvas (shown by Suspense when model is loading) */}
      {modelPath && <LoadingSpinner />}
    </div>
  )
}
```

Key points:
- `flat` prop disables renderer-level tone mapping (sets THREE.NoToneMapping). Tone mapping is handled exclusively in TrophyScene's EffectComposer as the last pass. This avoids double tone mapping per D-05/D-06/research.
- `dpr={[1, 2]}` caps pixel ratio at 2x per D-13.
- `gl={{ antialias: true }}` per D-14. No toneMapping or outputColorSpace in gl prop because `flat` handles tone mapping and R3F defaults outputColorSpace to SRGBColorSpace.
- `camera={{ position: [0, 1, 4], fov: 45 }}` provides a good initial viewing angle.
- When `modelPath` is null (no trophy selected), the Canvas renders empty (no scene content). This is intentional for the persistent Canvas pattern -- the Canvas stays mounted, only content changes.
- The LoadingSpinner is rendered as a DOM overlay (not inside Canvas) per the anti-pattern rule from research ("HTML UI inside Canvas via drei <Html> breaks accessibility").

NOTE: The LoadingSpinner approach needs refinement. The Suspense fallback inside Canvas can only render R3F components (not DOM). The DOM-based LoadingSpinner should use a state-based approach:
- Use drei's `useProgress` hook in a wrapper component to detect loading state
- Render the DOM LoadingSpinner conditionally based on loading progress
- The Suspense fallback inside Canvas should be `null` (nothing to render in 3D space during load)

**Create `src/components/LoadingSpinner.tsx`:**

A DOM-based loading overlay that uses drei's `useProgress` hook to track Three.js loader progress. Shows during .glb and .hdr loading, hides when complete.

```typescript
import { useProgress } from '@react-three/drei'

export function LoadingOverlay() {
  const { active, progress } = useProgress()

  if (!active) return null

  return (
    <div className="loading-overlay">
      <div className="loading-spinner" />
      <p className="loading-text">{Math.round(progress)}%</p>
    </div>
  )
}
```

NOTE: `useProgress` from drei works OUTSIDE the Canvas because it subscribes to a global loading manager. This is the correct pattern for DOM loading indicators with R3F.

**Update `src/App.tsx`:**

The App component implements the persistent Canvas pattern (D-03, D-04). The Canvas is mounted once at the root layout level. DOM content (pages, overlays) sits on top via CSS absolute positioning. React Router provides routing for the DOM layer.

import { useState } from 'react'
import { BrowserRouter, Routes, Route } from 'react-router'
import { TrophyViewer } from './viewer/TrophyViewer'
import { SHOWCASE_TROPHIES } from './data/showcase'
import type { MetalPreset } from './types'

function App() {
  const defaultTrophy = SHOWCASE_TROPHIES[0]
  const [activeModel, setActiveModel] = useState<string | null>(defaultTrophy?.modelPath ?? null)
  const [activePreset, setActivePreset] = useState<MetalPreset>(defaultTrophy?.materials.primary ?? 'polishedGold')

  return (
    <BrowserRouter>
      <div className="app-layout">
        {/* Persistent Canvas -- NEVER unmounts (D-03, D-04) */}
        <TrophyViewer modelPath={activeModel} metalPreset={activePreset} />

        {/* DOM overlay layer -- sits on top of Canvas */}
        <div className="dom-overlay">
          <Routes>
            <Route path="/" element={<div className="phase1-label">Thomas Lyte 3D Gallery</div>} />
            <Route path="/trophy/:slug" element={<div>Trophy Detail (Phase 2)</div>} />
          </Routes>
        </div>
      </div>
    </BrowserRouter>
  )
}

export default App
