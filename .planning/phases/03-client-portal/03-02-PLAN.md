1. Create src/pages/IterationDetail.tsx:

This page follows the same pattern as TrophyDetail.tsx but looks up an iteration from CLIENT_PROJECTS instead of SHOWCASE_TROPHIES.

Props interface:
```typescript
interface IterationDetailProps {
  onModelChange: (modelPath: string | null) => void
  onPresetChange: (preset: MetalPreset) => void
  onOriginalMaterialsChange: (value: boolean) => void
}
```

Component behavior:
- Extract route params: `const { code, slug } = useParams<{ code: string; slug: string }>()`
- Look up project: `const project = getProjectByCode(code ?? '')`
- Find iteration across all design options:
  ```typescript
  const iteration = project?.designOptions
    .flatMap(opt => opt.iterations)
    .find(iter => iter.slug === slug)
  ```
- If !project or !iteration: show "Iteration not found" (same .trophy-not-found pattern)
- Check auth: if localStorage `portal_auth_{code}` is NOT "true", redirect to `/project/${code}` (prevent direct URL access without auth). Use `const navigate = useNavigate()` and `useEffect` with `navigate(`/project/${code}`, { replace: true })`.
- On mount (when iteration found and authenticated): call onModelChange(iteration.modelPath), onPresetChange(iteration.materials.primary), onOriginalMaterialsChange(false) -- iterations use preset materials, not original
- On unmount: call onModelChange(null) (cleanup, same as TrophyDetail)
- Render:
  - Back link to `/project/${code}` with text "Back to Project" (same .back-link class)
  - Iteration info panel: reuse the pattern from TrophyInfo but render directly (not via TrophyInfo component, since Iteration has different shape than PublicTrophy). Show:
    - Iteration label (e.g., "v1 -- Initial concept") as the title in .trophy-info-name
    - Dimensions in .trophy-info-dimensions
    - Material in .trophy-info-material
  - ARButton with iteration.modelPath and iteration.usdzPath
  - All rendered in the DOM overlay (transparent, pointer-events auto on interactive elements)
